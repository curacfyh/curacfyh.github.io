基本功

面向对象的特征

  • 继承
  • 封装
  • 多态

final, finally, finalize 的区别

  • final：修饰类为不可继承；修饰变量为引用不可变；修饰方法为不可重写
  • finally：try-catch-finaly的用法
  • finalize：垃圾回收时会调用该方法

int 和 Integer 有什么区别

  • int为基本类型
  • Integer为int的封装类型

重载和重写的区别

  • 重载：相同方法名称但方法签名不同，方法实现可不同
  • 重写：对父类方法进行重定义

抽象类和接口有什么区别

  • 抽象类：不能多继承，可有非抽象方法，一般为实体的抽象
  • 接口：可多实现，可以有default方法，其余方法都为抽象方法，且默认为public，一般为行为的抽象

说说反射的用途及实现

  • 可以获得类、接口、对象的信息
  • 可以访问私有域

说说自定义注解的场景及实现

  • 譬如spring的@Bean，作用在工厂方法上时，标明该方法的返回值为容器中的一个实例。spring会在初始化容器时，获取注解标记过的方法，并调用工厂方法来获取实例

HTTP 请求的 GET 与 POST 方式的区别

  • 都是TCP请求
  • GET将请求参数附加到URL中的，不安全、幂等、可缓存、可书签，且为一次TCP请求并换回状态码200
  • POST除可在URL中附加参数外，数据主要放入请求体中，安全、非幂等、不可缓存和书签。为2次请求：1次请求header返回100，2次发送data返回200

session 与 cookie 区别

  • HTTP是无状态的，所以需要某种机制来标示具体用户
  • session是服务端的用户标识，具有唯一session id
  • cookie是从客户端传递用户标识到服务端
  • 客户端cookie如果被禁用，就需要url重写技术，在URL中加入sid=xxx来标识用户

session 分布式处理

  • 主流的是spring session的实现，并使用Redis来做session集中缓存处理
  • 也可以将数据库作为集中式session存储
  • 还可以使用session复制的方式，在web容器集群中复制session

JDBC 流程

  • 加载驱动
  • 获取连接
  • 开启事务
  • 获取statement
  • 执行sql
  • 提交事务

MVC 设计思想

  • 将表现view、控制器controller、模型model分离，各司其职
  • view负责客户端展现
  • control负责前端数据到后端服务的转发
  • model负责具体业务处理
